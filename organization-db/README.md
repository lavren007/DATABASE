# Проект: База данных структуры организации

## Описание проекта
Данный проект представляет собой базу данных для учёта сотрудников, отделов, ролей, проектов и задач. В рамках проекта разработана схема базы данных, скрипты для создания таблиц и наполнения тестовыми данными, а также выполнены три аналитических SQL-запроса с использованием рекурсивных CTE для извлечения иерархической информации.

Проект выполнен в рамках домашнего задания по дисциплине «Базы данных».

## Структура базы данных
Схема базы данных состоит из пяти таблиц:

### 1. `Departments`
Хранит информацию об отделах.

| Колонка         | Тип           | Описание                         |
|-----------------|---------------|----------------------------------|
| `DepartmentID`  | SERIAL        | Уникальный идентификатор отдела  |
| `DepartmentName`| VARCHAR(100)  | Название отдела                  |

### 2. `Roles`
Хранит информацию о ролях сотрудников.

| Колонка     | Тип           | Описание                         |
|-------------|---------------|----------------------------------|
| `RoleID`    | SERIAL        | Уникальный идентификатор роли    |
| `RoleName`  | VARCHAR(100)  | Название роли                    |

### 3. `Employees`
Хранит данные о сотрудниках.

| Колонка       | Тип           | Описание                                  |
|---------------|---------------|-------------------------------------------|
| `EmployeeID`  | SERIAL        | Уникальный идентификатор сотрудника       |
| `Name`        | VARCHAR(100)  | Имя сотрудника                            |
| `Position`    | VARCHAR(100)  | Должность                                 |
| `ManagerID`   | INT           | Ссылка на руководителя (внешний ключ)     |
| `DepartmentID`| INT           | Ссылка на отдел                           |
| `RoleID`      | INT           | Ссылка на роль                            |

### 4. `Projects`
Хранит информацию о проектах.

| Колонка       | Тип           | Описание                                  |
|---------------|---------------|-------------------------------------------|
| `ProjectID`   | SERIAL        | Уникальный идентификатор проекта          |
| `ProjectName` | VARCHAR(100)  | Название проекта                          |
| `StartDate`   | DATE          | Дата начала                               |
| `EndDate`     | DATE          | Дата окончания                            |
| `DepartmentID`| INT           | Ссылка на отдел, ответственный за проект  |

### 5. `Tasks`
Хранит информацию о задачах.

| Колонка       | Тип           | Описание                                  |
|---------------|---------------|-------------------------------------------|
| `TaskID`      | SERIAL        | Уникальный идентификатор задачи           |
| `TaskName`    | VARCHAR(100)  | Название задачи                           |
| `AssignedTo`  | INT           | Ссылка на сотрудника, ответственного      |
| `ProjectID`   | INT           | Ссылка на проект, к которому относится    |

## Инструкция по запуску

### Предварительные требования
- Установленный PostgreSQL (версии 12 или выше)
- Доступ к командной строке или клиенту базы данных (pgAdmin, DBeaver и т.д.)

### Шаги для развёртывания

1. **Создайте базу данных**
   ```sql
   CREATE DATABASE organization_db;
   ```

2. **Подключитесь к базе данных**
   ```bash
   psql -d organization_db -U your_username
   ```
   
3. **Выполните скрипт создания таблиц**
   ```bash
   \i schema.sql
   ```

4. **Наполните базу тестовыми данными**
   ```bash
   \i data.sql
   ```

5. **Выполните запросы из задач**
    - Для задачи 1: выполните содержимое `task1.sql`
    - Для задачи 2: выполните содержимое `task2.sql`
    - Для задачи 3: выполните содержимое `task3.sql`

### Ожидаемые результаты

Результаты соответствуют приведённым в условии задания (проверены на тестовых данных).

## Описание решений

### Задача 1
С помощью рекурсивного CTE строится иерархия подчинённых, начиная с Ивана Иванова (EmployeeID = 1). Затем для каждого сотрудника выбираются название отдела и роль. Для проектов учитываются все проекты отдела, в котором работает сотрудник (связь Projects → Departments). Задачи собираются через агрегацию по `AssignedTo`. Результат сортируется по имени сотрудника.

### Задача 2
Добавлены два поля:
- `TotalTasks` – количество задач, назначенных сотруднику (получено из подзапроса с COUNT).
- `TotalSubordinates` – количество прямых подчинённых (получено отдельным запросом, группирующим по `ManagerID`).
  Остальная логика аналогична задаче 1.

### Задача 3
Сначала рекурсивно для каждого сотрудника вычисляется количество всех его подчинённых (включая подчинённых подчинённых) с помощью CTE, которое затем агрегируется. Затем отбираются только сотрудники с ролью «Менеджер» и количеством подчинённых > 0. Для них выводятся проекты (через отдел) и задачи. Результат сортируется по имени.
