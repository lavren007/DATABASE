# Проект: База данных бронирования отелей

## Описание проекта
Данный проект представляет собой базу данных для учёта отелей, номеров, клиентов и бронирований. В рамках проекта разработана схема базы данных, скрипты для создания таблиц и наполнения тестовыми данными, а также выполнены три аналитических SQL-запроса для извлечения информации по заданным критериям.

Проект выполнен в рамках домашнего задания по дисциплине «Базы данных».

## Структура базы данных
Схема базы данных состоит из четырёх таблиц:

### 1. `Hotel`
Хранит информацию об отелях.

| Колонка     | Тип           | Описание                         |
|-------------|---------------|----------------------------------|
| `ID_hotel`  | SERIAL        | Уникальный идентификатор отеля   |
| `name`      | VARCHAR(255)  | Название отеля                   |
| `location`  | VARCHAR(255)  | Местоположение                   |

### 2. `Room`
Хранит данные о номерах.

| Колонка     | Тип           | Описание                                  |
|-------------|---------------|-------------------------------------------|
| `ID_room`   | SERIAL        | Уникальный идентификатор номера           |
| `ID_hotel`  | INT           | Ссылка на отель (внешний ключ)            |
| `room_type` | VARCHAR(20)   | Тип номера: Single, Double, Suite         |
| `price`     | DECIMAL(10,2) | Цена за ночь                              |
| `capacity`  | INT           | Вместимость                               |

### 3. `Customer`
Хранит информацию о клиентах.

| Колонка       | Тип           | Описание                         |
|---------------|---------------|----------------------------------|
| `ID_customer` | SERIAL        | Уникальный идентификатор клиента |
| `name`        | VARCHAR(255)  | Имя клиента                      |
| `email`       | VARCHAR(255)  | Электронная почта (уникальный)   |
| `phone`       | VARCHAR(20)   | Номер телефона                   |

### 4. `Booking`
Фиксирует бронирования.

| Колонка         | Тип    | Описание                         |
|-----------------|--------|----------------------------------|
| `ID_booking`    | SERIAL | Уникальный идентификатор брони   |
| `ID_room`       | INT    | Ссылка на номер                  |
| `ID_customer`   | INT    | Ссылка на клиента                |
| `check_in_date` | DATE   | Дата заезда                      |
| `check_out_date`| DATE   | Дата выезда                      |

## Инструкция по запуску

### Предварительные требования
- Установленный PostgreSQL (версии 12 или выше)
- Доступ к командной строке или клиенту базы данных (pgAdmin, DBeaver и т.д.)

### Шаги для развёртывания

1. **Создайте базу данных**
   ```sql
   CREATE DATABASE hotel_booking_db;
   ```

2. **Подключитесь к базе данных**
   ```bash
   psql -d hotel_booking_db -U your_username
   ```

3. **Выполните скрипт создания таблиц**
   ```bash
   \i schema.sql
   ```

4. **Наполните базу тестовыми данными**
   ```bash
   \i data.sql
   ```

5. **Выполните запросы из задач**
    - Для задачи 1: выполните содержимое `task1.sql`
    - Для задачи 2: выполните содержимое `task2.sql`
    - Для задачи 3: выполните содержимое `task3.sql`

## Описание решений

### Задача 1
С помощью общих табличных выражений (CTE) группируем бронирования по клиентам, вычисляем количество бронирований, количество уникальных отелей, общее количество дней проживания и формируем список отелей через `STRING_AGG`. Отбираем клиентов с количеством бронирований >2 и более чем одним уникальным отелем. Средняя длительность пребывания вычисляется как общее количество дней, делённое на количество бронирований. Результат сортируется по убыванию количества бронирований.

### Задача 2
Сначала вычисляем для каждого клиента общее количество бронирований, количество уникальных отелей и общую сумму, потраченную на бронирования (суммируем цены номеров). Отбираем клиентов с количеством бронирований >2, количеством уникальных отелей >1 и общей суммой >500. Выводим поля согласно условию. Сортировка по возрастанию общей суммы.

### Задача 3
Сначала определяем категорию каждого отеля на основе средней стоимости номера: дешевый (<175), средний (175–300), дорогой (>300). Затем для каждого клиента собираем все посещённые им отели с их категориями. Далее каждому клиенту присваиваем предпочтительную категорию по правилу: если есть дорогой отель → «Дорогой», иначе если есть средний → «Средний», иначе «Дешевый». Формируем список уникальных отелей через `STRING_AGG`. Сортировка выполняется по категории в порядке: дешевый, средний, дорогой.
