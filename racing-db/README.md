# Проект: База данных автомобильных гонок

## Описание проекта
Данный проект представляет собой базу данных для учёта классов автомобилей, самих автомобилей, гоночных событий и результатов гонок. В рамках проекта разработана схема базы данных, скрипты для создания таблиц и наполнения тестовыми данными, а также выполнены пять аналитических SQL-запросов для извлечения информации по заданным критериям.

Проект выполнен в рамках домашнего задания по дисциплине «Базы данных».

## Структура базы данных
Схема базы данных состоит из четырёх таблиц:

### 1. `Classes`
Хранит информацию о классах автомобилей.

| Колонка       | Тип           | Описание                                    |
|---------------|---------------|---------------------------------------------|
| `class`       | VARCHAR(100)  | Название класса (первичный ключ)            |
| `type`        | VARCHAR(20)   | Тип класса: Racing, Street                  |
| `country`     | VARCHAR(100)  | Страна производства                         |
| `numDoors`    | INT           | Количество дверей                           |
| `engineSize`  | DECIMAL(3,1)  | Объём двигателя (л)                         |
| `weight`      | INT           | Вес автомобиля (кг)                         |

### 2. `Cars`
Хранит данные об автомобилях.

| Колонка       | Тип           | Описание                                    |
|---------------|---------------|---------------------------------------------|
| `name`        | VARCHAR(100)  | Название автомобиля (первичный ключ)        |
| `class`       | VARCHAR(100)  | Ссылка на класс из `Classes`                |
| `year`        | INT           | Год выпуска                                 |

### 3. `Races`
Хранит информацию о гонках.

| Колонка       | Тип           | Описание                                    |
|---------------|---------------|---------------------------------------------|
| `name`        | VARCHAR(100)  | Название гонки (первичный ключ)             |
| `date`        | DATE          | Дата проведения                             |

### 4. `Results`
Фиксирует результаты участия автомобилей в гонках.

| Колонка       | Тип           | Описание                                    |
|---------------|---------------|---------------------------------------------|
| `car`         | VARCHAR(100)  | Название автомобиля (внешний ключ)          |
| `race`        | VARCHAR(100)  | Название гонки (внешний ключ)               |
| `position`    | INT           | Занятое место                               |

## Инструкция по запуску

### Предварительные требования
- Установленный PostgreSQL (версии 12 или выше)
- Доступ к командной строке или клиенту базы данных (pgAdmin, DBeaver и т.д.)

### Шаги для развёртывания

1. **Создайте базу данных**
   ```sql
   CREATE DATABASE racing_db;
   ```

2. **Подключитесь к базе данных**
   ```bash
   psql -d racing_db -U your_username
   ```

3. **Выполните скрипт создания таблиц**
   ```bash
   \i schema.sql
   ```
   Или скопируйте содержимое `schema.sql` в клиент БД и выполните.

4. **Наполните базу тестовыми данными**
   ```bash
   \i data.sql
   ```

5. **Выполните запросы из задач**
    - Для задачи 1: выполните содержимое `task1.sql`
    - Для задачи 2: выполните содержимое `task2.sql`
    - Для задачи 3: выполните содержимое `task3.sql`
    - Для задачи 4: выполните содержимое `task4.sql`
    - Для задачи 5: выполните содержимое `task5.sql`

## Описание решений

### Задача 1
Для каждого автомобиля вычисляется средняя позиция и количество гонок. Затем для каждого класса находится минимальная средняя позиция среди его автомобилей. Выбираются те автомобили, чья средняя позиция равна минимальной для своего класса. Результат сортируется по средней позиции.

### Задача 2
Вычисляется средняя позиция для каждого автомобиля, затем выбирается запись с наименьшим значением. При равенстве средних позиций выбирается автомобиль с именем, идущим первым по алфавиту. К результату добавляется страна производства из таблицы `Classes`.

### Задача 3
Сначала для каждого класса вычисляется средняя позиция как среднее арифметическое средних позиций всех его автомобилей. Определяется минимальное значение среди этих средних. Все классы, достигшие этого минимума, считаются целевыми. Для каждого автомобиля из этих классов выводится его средняя позиция, количество гонок, страна класса и общее количество уникальных гонок, в которых участвовали автомобили этого класса.

### Задача 4
Для каждого класса, в котором зарегистрировано не менее двух автомобилей, вычисляется средняя позиция по классу (среднее арифметическое средних позиций автомобилей). Затем выбираются автомобили, чья средняя позиция строго меньше средней по классу. Результат сортируется по классу и по средней позиции.

### Задача 5
Для каждого класса подсчитывается количество автомобилей, у которых средняя позиция больше 3.0. Определяются классы, имеющие максимальное такое количество. Для всех автомобилей из этих классов выводится их средняя позиция, количество гонок, страна класса, общее количество гонок класса и значение `low_position_count` (количество автомобилей в классе с позицией >3.0). Сортировка выполняется по убыванию `low_position_count`, затем по классу и по средней позиции.
