
# Проект: База данных транспортных средств

## Описание проекта
Данный проект представляет собой базу данных для учёта транспортных средств трёх типов: автомобили, мотоциклы и велосипеды. В рамках проекта разработана схема базы данных, скрипты для создания таблиц и наполнения тестовыми данными, а также выполнены два аналитических SQL-запроса для извлечения информации по заданным критериям.

Проект выполнен в рамках домашнего задания по дисциплине «Базы данных».

## Структура базы данных
Схема базы данных состоит из четырёх таблиц:

### 1. `Vehicle` (родительская таблица)
Хранит общую информацию о моделях транспорта.

| Колонка | Тип | Описание |
|---------|------|----------|
| `maker` | VARCHAR(100) | Производитель |
| `model` | VARCHAR(100) | Название модели (первичный ключ) |
| `type` | ENUM / VARCHAR | Тип транспорта: Car, Motorcycle, Bicycle |

### 2. `Car`
Хранит данные об автомобилях.

| Колонка | Тип | Описание |
|---------|------|----------|
| `vin` | VARCHAR(17) | Уникальный идентификатор (VIN) |
| `model` | VARCHAR(100) | Ссылка на модель из `Vehicle` |
| `engine_capacity` | DECIMAL(4,2) | Объём двигателя (л) |
| `horsepower` | INT | Мощность (л.с.) |
| `price` | DECIMAL(10,2) | Цена ($) |
| `transmission` | ENUM/VARCHAR | Тип КПП (Automatic/Manual) |

### 3. `Motorcycle`
Хранит данные о мотоциклах.

| Колонка | Тип | Описание |
|---------|------|----------|
| `vin` | VARCHAR(17) | Уникальный идентификатор (VIN) |
| `model` | VARCHAR(100) | Ссылка на модель из `Vehicle` |
| `engine_capacity` | DECIMAL(4,2) | Объём двигателя (л) |
| `horsepower` | INT | Мощность (л.с.) |
| `price` | DECIMAL(10,2) | Цена ($) |
| `type` | ENUM/VARCHAR | Тип мотоцикла (Sport/Cruiser/Touring) |

### 4. `Bicycle`
Хранит данные о велосипедах.

| Колонка | Тип | Описание |
|---------|------|----------|
| `serial_number` | VARCHAR(20) | Серийный номер (первичный ключ) |
| `model` | VARCHAR(100) | Ссылка на модель из `Vehicle` |
| `gear_count` | INT | Количество передач |
| `price` | DECIMAL(10,2) | Цена ($) |
| `type` | ENUM/VARCHAR | Тип велосипеда (Mountain/Road/Hybrid) |

## Инструкция по запуску

### Предварительные требования
- Установленный PostgreSQL (версии 12 или выше) или MySQL (версии 8 или выше)
- Доступ к командной строке или клиенту базы данных (pgAdmin, DBeaver, MySQL Workbench и т.д.)

### Шаги для развёртывания

1. **Создайте базу данных**
   ```sql
   CREATE DATABASE transport_db;
   ```

2. **Подключитесь к базе данных**
   ```bash
   psql -d transport_db -U your_username
   ```

3. **Выполните скрипт создания таблиц**
   ```bash
   \i schema.sql
   ```
   Или скопируйте содержимое `schema.sql` в клиент БД и выполните.

4. **Наполните базу тестовыми данными**
   ```bash
   \i data.sql
   ```

5. **Выполните запросы из задач**
    - Для задачи 1: выполните содержимое `task1.sql`
    - Для задачи 2: выполните содержимое `task2.sql`

## Описание решений

### Задача 1
Выбраны мотоциклы с мощностью более 150 л.с., ценой менее 20 000 долларов и типом «Sport». Результат отсортирован по убыванию мощности. Используется соединение таблиц `Motorcycle` и `Vehicle` для получения названия производителя.

### Задача 2
Выполнено объединение трёх выборок через `UNION`:
- **Автомобили**: мощность >150 л.с., объём <3 л, цена <35 000$.
- **Мотоциклы**: мощность >150 л.с., объём <1.5 л, цена <20 000$.
- **Велосипеды**: количество передач >18, цена <4 000$. Для велосипедов в колонки `horsepower` и `engine_capacity` подставляется `NULL`.

Результаты отсортированы по мощности в порядке убывания; велосипеды (с `NULL`) размещены в конце с помощью `NULLS LAST`.
